# Reflection

## Skill Role

这是公用的内部质检能力，用于在图片生成后检查结果是否满足原始需求。它不是用户可直接点名的普通能力，而是生成流程里的自我评审与重试决策环节。

## Use When

- 上一步图片生成返回了 `needs_reflection=true`。
- 运行时提供了 `reflection_context`，其中包含生成图片路径、原始提示词、当前提示词、参考图和重试次数。
- 需要判断图片是否足够满足需求，以及是否值得再次生成。

## Do Not Use When

- 用户直接发送普通消息要求“反思一下”“评价一下”；这类请求应由 `general` 或 `image_understanding` 处理。
- 没有生成图片路径或原始需求上下文。
- 已达到最大重试次数，除非调用方仍明确要求质检文本。

## Review Criteria

- 内容准确性：主体、文字、元素、风格是否符合请求。
- 构图与版式：视觉重心、留白、信息层级是否合理。
- 色彩与质感：色彩情绪、光影、材质表现是否稳定。
- 参考一致性：参考图的关键身份、品牌感、主体特征是否被保留。
- 生成质量：是否有明显变形、伪影、错字、缺失或不自然内容。

## Output Contract

返回重试决策、用户可读的简短复盘文本、必要时返回优化后的提示词。不要直接发送图片，不要修改文件。
