# Image Understanding

## Skill Role

这是公用的视觉理解能力，负责“看图说话”。它读取用户提供的图片，输出解释、判断、设计点评、差异比较或改进建议，但不生成新图片，也不改动原图。

## Use When

- 用户发送图片并问“这是什么”“帮我看看”“评价一下”“哪里有问题”。
- 用户要求分析构图、配色、字体、层级、风格、品牌感或可读性。
- 用户发送多张图片，要求对比、选优、找差异、总结共同点。
- 用户要求从图片里提取可见信息，但没有要求直接修改图片。

## Do Not Use When

- 用户要求生成新图片、重绘、风格化、换背景、改版式或调整画面；使用 `image_gen`。
- 用户只是想轻松吐槽图片；使用 `funny`。
- 用户没有图片，且只是普通文字咨询；使用 `general`。
- 用户要求对文件内容做结构化处理，而不是单纯看图片。

## Analysis Guidance

- 单图默认从构图、色彩、字体、层级、信息清晰度、整体风格给出判断。
- 多图默认比较视觉语言、优缺点、适用场景和可借鉴点。
- 如果用户有具体问题，优先回答问题，不强行套完整设计检查清单。
- 对不确定内容使用“看起来像”“可能是”，不要编造看不见的信息。
- 给建议时尽量具体到可改动作，例如“标题字号降低一档”“提高正文对比度”。

## Output Contract

返回纯文本分析。不要输出图片路径，不要触发图片生成。
